hub – Discover and Load Motors¶

connect(device_type='USB2Serial', serial_id=None, port_path=None, baudrate=1000000, timeout=50, full_ram=False, verbose=True, motor_range=(0, 253), enable_pyftdi=True, start=True, latency=1)¶

Guess interface, scans it and configure controller.

Parameters:
  • device_type – kind of device for sending messages to the motors. can be either of ‘USB2Dynamixel’, ‘USB2Serial’, ‘USB2AX’, ‘CM-5’, ‘CM-510’, ‘CM-513’, ‘CM-700’, ‘CM-900’, ‘OpenCM9.04’, ‘CM-100’, ‘CM-100A’, ‘SerialPort’.
  • serial_id – if multiple usb/serial adaptaters are present, this parameter can connect to a specific one.
  • port_path –

    an explicit /dev/ path to a device.

    name of connection type, see controller for supported connection types
disconnect(uid)¶

Disconnect and delete a controller

:param uid the index of the controller

controller(uid)¶

Return the controller with the given uid.
